# v1.0 Demo Walkthrough

This walkthrough is the clean reviewer path for the release demo.

## 1. Clean Setup

```bash
git clone https://github.com/kashyapHebbar/autonomous-enterprise-ai-os.git
cd autonomous-enterprise-ai-os
python3.11 -m venv .venv
source .venv/bin/activate
make install
```

## 2. Run Validation

```bash
make lint
make smoke
make k8s-validate
make cloud-validate
```

Run the full test suite when time allows:

```bash
make test
```

## 3. Run The Procurement Demo

```bash
make demo
```

The demo uses `examples/procurement_demo.csv` and writes artifacts under
`artifacts/procurement_demo/<run_id>/`.

Expected terminal output:

- Run ID
- Status
- Trace ID
- Artifact root
- Summary path
- Metrics path
- Event count
- Dashboard, report, evaluation, KPI, chart, and code artifact paths

## 4. Inspect Demo Files

Open the generated `demo_summary.json`. It records:

- Run status and trace ID
- Dataset path
- Artifact metadata
- Evaluation pass/fail score and checks
- Event count
- Metrics file path

Open the generated dashboard and report artifacts from the printed paths.

## 5. Start The API/UI

```bash
make dev
```

Open:

- Run control plane: `http://127.0.0.1:8000/app`
- Artifact browser: `http://127.0.0.1:8000/app/artifacts`
- Admin UI: `http://127.0.0.1:8000/app/admin`
- API docs: `http://127.0.0.1:8000/docs`
- Metrics: `http://127.0.0.1:8000/metrics`

If you import or recreate a run through the API, open:

```text
http://127.0.0.1:8000/run-inspector/runs/{run_id}
```

## 6. UI Review Script

Use this order for a short portfolio walkthrough:

1. `/app`: show run creation and recent run list.
2. `/app/admin`: show registered agents, connector health, credential profiles, and policy rules.
3. `/app/artifacts`: show dashboard/report previews, artifact grouping, lineage, and safe downloads.
4. `/run-inspector/runs/{run_id}`: show graph nodes, timeline, events, approvals, evaluations, and
   deployment history.
5. `/metrics`: show Prometheus metrics generated by repository state.

## 7. Optional Deployment Review

For production-shaped deployment review:

```bash
make k8s-validate
make cloud-validate
```

Then inspect:

- `deploy/kubernetes/README.md`
- `deploy/kubernetes/overlays/production`
- `deploy/cloud/aws/README.md`
- `deploy/cloud/aws/terraform`
